# Issue and Pull Request Usage Guidelines

## Issues

- Use the matching template for all new work.
- Apply one type label, one priority label, and the relevant domain label.
- Epics should link child work and exit criteria.
- Security issues should avoid sensitive public detail.

## Pull Requests

- Select the correct template for feature, fix, hotfix, or docs work.
- Keep PRs scoped to one logical chunk.
- Include validation evidence and documentation impact.
- Link the issue and ADR where applicable.
