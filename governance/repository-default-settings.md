# Repository Default Settings

Recommended defaults for all InfraLynx repositories:

- Default branch: `develop` during active pre-release delivery
- Merge strategy: squash merge enabled
- Merge commit: disabled
- Rebase merge: disabled unless repository-specific tooling requires it
- Automatically delete head branches after merge: enabled
- Discussions: optional per repository
- Wikis: disabled
- Projects: enabled
- Issues: enabled
- Vulnerability alerts: enabled

These defaults should be reviewed again when release management becomes active.
